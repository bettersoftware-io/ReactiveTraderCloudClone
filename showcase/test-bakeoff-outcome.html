Test-tooling bake-off · concluded 2026-07-20

We shipped every tool to judge them.
Then we kept them all picked one each.

The repo ran multiple testing tools per category on purpose — to evaluate them on real evidence, then choose a single winner and retire the rest. Here's how it ended: what won, what was cut, and the proof that nothing was lost.

Verdicts — pick the winner, retire the rest

Visual diff 3 tiers → 1

KEEP
playwright — plain Playwright over a Vite host. Won as the cross-framework portability contract (one spec, both clients), not on speed — it was the slowest of the three.
DELETE
playwright-ct — its Solid side was never a real CT mount; the adapter trailed core by ~1.5 yr.
COVERAGE
vitest-browser — assert retired, kept coverage-only: renders every scenario for istanbul, pixel assert compiled out.

Browser e2e Cypress out, Gherkin parked

KEEP
native Playwright — react + solid; declared the browser source-of-truth.
DELETE
cypress ×2 — forked queue-aware scenario layer was the cost; arm64/Electron hazard.
PARK
playwright-cucumber ×2 — off the PR gate, run weekly so the feature tree can't rot.

Presenter 4 peers → 1

KEEP
vitest-fake-timers — plain vitest, fake timers. Fast and slim.
DELETE
cucumber · cucumber-fake-timers · vitest-quickpickle-fake-timers

Dropped @sinonjs/fake-timers, quickpickle, and the RealAwaitHelpers real-timer helper.

Contract tier untouched

KEEP
Its four "variants" were never a bake-off — {react,solid} × {plain,coverage} over one 86-file spec corpus. Parity insurance + the ≥95% coverage gates. Left exactly as-is.

The part worth celebrating — nothing was lost

Coverage integrity, verified adversarially (not assumed)

  • All 48 deleted Cypress it()s map name-for-name to surviving Playwright tests — the two differences both in Playwright's favour.
  • Deleted presenter step files held zero expect/assert calls — drivers, not oracles.
  • The visual matrix stayed at 1282 scenarios; the surviving golden tree is byte-identical at 2,659 PNGs.
  • A real gap was found and closed: admin/incident's @presenter scenario had been cucumber-only and invisible to gate 21 — recovered into the surviving peer (21 → 22) and the gate extended to enforce it. The retirement made the suite more correct.

Two reductions accepted on purpose — not regressions

  • A Gherkin / step-tree regression is now caught weekly, not per-PR (the parked browser layer).
  • No presenter test runs under real timers any more — a bug that only shows up with a real setTimeout has lost its witness.
