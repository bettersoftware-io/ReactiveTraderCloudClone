# Phase 1: Layout Engine (LayoutPort) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hand-rolled `shell/layout/Workspace.tsx` flex layout with an in-house, prototype-faithful layout engine behind a replaceable `LayoutPort` seam. The engine renders a tree of resizable split panes and reproduces the prototype's four behaviors exactly: resizable drag-handle splits, maximize (siblings collapse to strips), collapse-to-strip, and a pinned bottom blotter slot. The app references panels only by `PanelId` through a `PanelRegistry`; swapping to a `DockviewLayoutEngine` later satisfies the same consumption contract with no module changes.

**Architecture:** Three layers, dependency rule unchanged (`domain` ← `app` ← `ui`):
- `app/layout/layoutPort.ts` — the `LayoutPort` seam + `LayoutState`/`LayoutNode`/`PanelSpec`/`PanelId`/`SplitDir` types. **App layer, NOT `@rtc/domain`.** Rationale (interfaces doc §5): layout is *presentation infrastructure*, not business domain, and `@rtc/domain` is rxjs-only/business-pure — a layout tree with split dirs and pixel sizes has no place there. The dependency-cruiser `domain-stays-pure` rule would also reject it.
- `app/presenters/LayoutMachine.ts` — `createLayoutMachine(port): Machine<LayoutState, LayoutIntents>`, a neutral VM holding the tree and applying ops. Fully unit-tested, no DOM. Follows the `NotionalMachine` intent-driven reducer precedent (Subjects → `scan` → `state(stream$, initial)` + warm subscription torn down in `dispose()`).
- `ui/shell/layout/engine/InhouseLayoutEngine.tsx` — dumb renderer of `LayoutState`. The resize-handle pointer-event drag and the FLIP/strip transitions are **the one framework-coupled spot** (interfaces doc §5 / spec §Phase 1), allowed here behind the port — the single place a Solid swap re-implements. Co-located `.module.css` with semantic `data-*` state + `--custom-property` geometry for split sizes.

**Tech Stack:** TypeScript, RxJS (`@rx-state/core` `state()`), `@react-rxjs/core` (`useStateObservable` via the existing `useMachine` bridge only), React 19, Vitest (unit), Vitest+RTL (UI-contract via the `@ui-contract` harness), Playwright-CT (visual goldens). CSS Modules + stylelint `declaration-strict-value`. No new runtime dependency (pointer events + Web Animations are platform APIs; Motion One arrives in Phase 0, not used here).

### Task 2 — `LayoutMachine` (neutral VM + intents)

`createLayoutMachine(port)` holds the tree and applies the five intents over an immutable reducer. Follows the `NotionalMachine` precedent exactly: intent Subjects → merged event stream → `scan(reducer, initial)` → `state(stream$, initial)` + a warm subscription released in `dispose()`.

**Files:**
- Create: `packages/client-react/src/app/presenters/LayoutMachine.ts`
- Test: `packages/client-react/src/app/presenters/__tests__/LayoutMachine.test.ts`

**Interfaces:**
- Consumes (interfaces doc §5): `LayoutPort`, `LayoutState`, `LayoutNode`, `PanelId` from `#/app/layout/layoutPort`; `Machine<TState, TIntents>` from `#/app/presenters/machine`.
- Produces (exactly, interfaces doc §5):
  ```typescript
  export interface LayoutIntents {
    maximize(id: PanelId): void;
    restore(): void;
    collapse(id: PanelId): void;
    expand(id: PanelId): void;
    resize(path: readonly number[], sizes: readonly number[]): void;
  }
  export function createLayoutMachine(port: LayoutPort): Machine<LayoutState, LayoutIntents>;
  ```

**Reducer semantics (pinned so the engine and goldens are deterministic):**
- `maximize(id)` → `{ ...state, maximized: id }` (engine renders the maximized panel full-bleed; every *other* panel becomes a strip — "maximize-collapses-siblings-to-strips"). No-op if `id` is not a panel in the tree.
- `restore()` → `{ ...state, maximized: null }`.
- `collapse(id)` → add `id` to `collapsed` (deduped) → engine renders that panel as a strip.
- `expand(id)` → remove `id` from `collapsed`.
- `resize(path, sizes)` → walk `root` by `path` (each index selects a `split` child) to the target `split` node and replace its `sizes` immutably; out-of-range path or non-split target is a no-op (defensive). A `pinned` panel's slot is never part of a resizable split's `sizes` (pinned lives in a fixed bottom slot — see Task 4), so resizing never touches it.

  ```typescript
  import { type DefaultedStateObservable, state } from "@rx-state/core";
  import { merge, Subject } from "rxjs";
  import { map, scan } from "rxjs/operators";

  import type {
    LayoutNode,
    LayoutPort,
    LayoutState,
    PanelId,
  } from "#/app/layout/layoutPort";

  import type { Machine } from "./machine";

  export interface LayoutIntents {
    maximize(id: PanelId): void;
    restore(): void;
    collapse(id: PanelId): void;
    expand(id: PanelId): void;
    resize(path: readonly number[], sizes: readonly number[]): void;
  }

  type LayoutEvent =
    | { type: "maximize"; id: PanelId }
    | { type: "restore" }
    | { type: "collapse"; id: PanelId }
    | { type: "expand"; id: PanelId }
    | { type: "resize"; path: readonly number[]; sizes: readonly number[] };

  /** Replace the `sizes` of the split node reached by walking `path` from `node`.
   * Each path index selects a split child; a non-split target or an out-of-range
   * index returns the node unchanged (defensive no-op). Pure + immutable. */
  function resizeAt(
    node: LayoutNode,
    path: readonly number[],
    sizes: readonly number[],
  ): LayoutNode {
    if (node.kind !== "split") return node;
    if (path.length === 0) return { ...node, sizes };
    const [head, ...rest] = path;
    if (head < 0 || head >= node.children.length) return node;
    const child = node.children[head];
    const nextChild = resizeAt(child, rest, sizes);
    if (nextChild === child) return node;
    const children = node.children.map((c, i) => (i === head ? nextChild : c));
    return { ...node, children };
  }

  function reduce(state: LayoutState, event: LayoutEvent): LayoutState {
    switch (event.type) {
      case "maximize":
        return { ...state, maximized: event.id };
      case "restore":
        return { ...state, maximized: null };
      case "collapse":
        return state.collapsed.includes(event.id)
          ? state
          : { ...state, collapsed: [...state.collapsed, event.id] };
      case "expand":
        return {
          ...state,
          collapsed: state.collapsed.filter((id) => id !== event.id),
        };
      case "resize":
        return { ...state, root: resizeAt(state.root, event.path, event.sizes) };
    }
  }

  /** Neutral layout view-model. Holds the tree, applies the five intents over an
   * immutable reducer, and emits LayoutState. No DOM. Mirrors the NotionalMachine
   * intent-driven precedent: Subjects → merged events → scan → state() + a warm
   * subscription released in dispose(). */
  export function createLayoutMachine(
    port: LayoutPort,
  ): Machine<LayoutState, LayoutIntents> {
    const maximize$ = new Subject<PanelId>();
    const restore$ = new Subject<void>();
    const collapse$ = new Subject<PanelId>();
    const expand$ = new Subject<PanelId>();
    const resize$ = new Subject<{
      path: readonly number[];
      sizes: readonly number[];
    }>();

    const events$ = merge(
      maximize$.pipe(map((id): LayoutEvent => ({ type: "maximize", id }))),
      restore$.pipe(map((): LayoutEvent => ({ type: "restore" }))),
      collapse$.pipe(map((id): LayoutEvent => ({ type: "collapse", id }))),
      expand$.pipe(map((id): LayoutEvent => ({ type: "expand", id }))),
      resize$.pipe(
        map(({ path, sizes }): LayoutEvent => ({ type: "resize", path, sizes })),
      ),
    );

    const stream$ = events$.pipe(scan(reduce, port.initial));

    const state$: DefaultedStateObservable<LayoutState> = state(
      stream$,
      port.initial,
    );

    // Keep state$ warm so it carries its default before useMachine first renders.
    const warm = state$.subscribe();

    return {
      state$,
      intents: {
        maximize: (id: PanelId) => maximize$.next(id),
        restore: () => restore$.next(),
        collapse: (id: PanelId) => collapse$.next(id),
        expand: (id: PanelId) => expand$.next(id),
        resize: (path: readonly number[], sizes: readonly number[]) =>
          resize$.next({ path, sizes }),
      },
      dispose: () => {
        maximize$.complete();
        restore$.complete();
        collapse$.complete();
        expand$.complete();
        resize$.complete();
        warm.unsubscribe();
      },
    };
  }
  ```
  > Note: `scan` over `port.initial` plus `state(stream$, port.initial)` matches the NotionalMachine idiom (initial supplied to both, so the synchronous default is present before the first event). The reducer is exhaustive over `LayoutEvent`, satisfying ESLint `no-restricted-syntax`/typed-switch checks without a `default` arm.

### Task 4 — `InhouseLayoutEngine` (dumb renderer) + `PanelRegistry`

- A split node renders children inline-styled with the geometry custom property `style={{ "--split-size": size } as CSSProperties}` per child (flex-basis driven from it in CSS), `data-dir={dir}`. Between adjacent children a drag handle `data-testid={`handle-${pathKey}-${i}`}` (where `pathKey` is the dot-joined path) with `role="separator"`, `data-orientation={dir === "row" ? "vertical" : "horizontal"}`.
- **Drag:** `onPointerDown` on a handle captures the pointer (`setPointerCapture`), tracks `pointermove` deltas against the split's bounding box, computes the two neighbouring fractions, and calls `onResize(path, nextSizes)` live; `pointerup` releases capture. This is the framework-coupled code; it lives only here.
- **Maximize:** when `state.maximized === id`, render that panel full-bleed (`data-maximized="true"`) and render every other panel as a strip (`data-strip="true"`, body hidden, click-to-restore on the strip).
- **Pinned:** a panel whose `specs[panelId].pinned` is true and that sits as the last child of the root column split renders in a fixed bottom slot (`data-pinned="true"`); it is never given a drag handle.

  ```tsx
  import {
    type CSSProperties,
    type PointerEvent as ReactPointerEvent,
    type ReactElement,
    useRef,
  } from "react";

  import type {
    LayoutNode,
    LayoutState,
    PanelId,
    PanelSpec,
  } from "#/app/layout/layoutPort";
  import { PANEL_SPECS } from "#/app/layout/defaultLayoutPort";
  import type { LayoutIntents } from "#/app/presenters/LayoutMachine";

  import type { PanelRegistry } from "./panelRegistry";

  import styles from "./InhouseLayoutEngine.module.css";

  export interface InhouseLayoutEngineProps {
    state: LayoutState;
    registry: PanelRegistry;
    specs?: Readonly<Record<PanelId, PanelSpec>>;
    onMaximize: LayoutIntents["maximize"];
    onRestore: LayoutIntents["restore"];
    onCollapse: LayoutIntents["collapse"];
    onExpand: LayoutIntents["expand"];
    onResize: LayoutIntents["resize"];
  }

  /** Dumb renderer of a LayoutState. The pointer-event resize drag and the
   * strip/maximize transitions are the ONE framework-coupled spot in the app
   * (interfaces doc §5) — confined to this component behind the LayoutPort, so a
   * SolidJS swap re-implements only this file. No rxjs/localStorage/fetch here;
   * geometry is a `--split-size` custom property, state is semantic data-*. */
  export function InhouseLayoutEngine({
    state,
    registry,
    specs = PANEL_SPECS,
    onMaximize,
    onRestore,
    onCollapse,
    onExpand,
    onResize,
  }: InhouseLayoutEngineProps): ReactElement {
    const isMaximized = state.maximized !== null;

    function renderPanel(panelId: PanelId): ReactElement {
      const spec = specs[panelId];
      const collapsed = state.collapsed.includes(panelId);
      const maximizedHere = state.maximized === panelId;
      const strip = (isMaximized && !maximizedHere) || collapsed;
      return (
        <section
          key={panelId}
          data-testid={`panel-${panelId}`}
          className={styles.panel}
          data-collapsed={collapsed ? "true" : "false"}
          data-pinned={spec?.pinned ? "true" : "false"}
          data-strip={strip ? "true" : "false"}
          data-maximized={maximizedHere ? "true" : "false"}
        >
          <header
            data-testid={`panel-${panelId}-header`}
            className={styles.panelHeader}
          >
            <span className={styles.panelTitle}>{spec?.title ?? panelId}</span>
            <div className={styles.panelControls}>
              <button
                type="button"
                data-testid={`panel-${panelId}-collapse`}
                className={styles.panelControl}
                aria-label={collapsed ? `Expand ${spec?.title ?? panelId}` : `Collapse ${spec?.title ?? panelId}`}
                onClick={() => (collapsed ? onExpand(panelId) : onCollapse(panelId))}
              >
                {collapsed ? "▢" : "—"}
              </button>
              <button
                type="button"
                data-testid={`panel-${panelId}-maximize`}
                className={styles.panelControl}
                aria-label={maximizedHere ? `Restore ${spec?.title ?? panelId}` : `Maximize ${spec?.title ?? panelId}`}
                onClick={() => (maximizedHere ? onRestore() : onMaximize(panelId))}
              >
                {maximizedHere ? "▣" : "▢"}
              </button>
            </div>
          </header>
          {strip ? null : (
            <div className={styles.panelBody}>{registry[panelId]?.()}</div>
          )}
        </section>
      );
    }

    function renderNode(
      node: LayoutNode,
      path: readonly number[],
    ): ReactElement {
      if (node.kind === "panel") return renderPanel(node.panelId);

      const pathKey = path.join(".");
      const splitRef = useRef<HTMLDivElement | null>(null);

      function onHandlePointerDown(
        index: number,
        e: ReactPointerEvent<HTMLDivElement>,
      ): void {
        e.preventDefault();
        const handle = e.currentTarget;
        handle.setPointerCapture(e.pointerId);
        const container = splitRef.current;
        if (!container) return;
        const rect = container.getBoundingClientRect();
        const total = node.dir === "row" ? rect.width : rect.height;
        const origin = node.dir === "row" ? rect.left : rect.top;
        const a = node.sizes[index];
        const b = node.sizes[index + 1];
        const pair = a + b;

        function move(ev: PointerEvent): void {
          const pos = (node.dir === "row" ? ev.clientX : ev.clientY) - origin;
          const before = node.sizes
            .slice(0, index)
            .reduce((s, v) => s + v, 0);
          let fracA = pos / total - before;
          fracA = Math.max(0.05, Math.min(pair - 0.05, fracA));
          const next = node.sizes.slice();
          next[index] = fracA;
          next[index + 1] = pair - fracA;
          onResize(path, next);
        }
        function up(ev: PointerEvent): void {
          handle.releasePointerCapture(ev.pointerId);
          handle.removeEventListener("pointermove", move);
          handle.removeEventListener("pointerup", up);
        }
        handle.addEventListener("pointermove", move);
        handle.addEventListener("pointerup", up);
      }

      // A pinned last child of a column split renders in the fixed bottom slot.
      const lastIsPinned =
        node.dir === "column" &&
        (() => {
          const last = node.children[node.children.length - 1];
          return last.kind === "panel" && specs[last.panelId]?.pinned === true;
        })();

      return (
        <div
          key={pathKey || "root"}
          ref={splitRef}
          className={styles.split}
          data-dir={node.dir}
          data-pinned-tail={lastIsPinned ? "true" : "false"}
        >
          {node.children.map((child, i) => {
            const childPinned =
              child.kind === "panel" && specs[child.panelId]?.pinned === true;
            return (
              <div
                key={`${pathKey}.${i}`}
                className={styles.cell}
                data-pinned-cell={childPinned ? "true" : "false"}
                style={
                  childPinned
                    ? undefined
                    : ({ "--split-size": String(node.sizes[i]) } as CSSProperties)
                }
              >
                {renderNode(child, [...path, i])}
                {!childPinned &&
                i < node.children.length - 1 &&
                !(
                  node.children[i + 1].kind === "panel" &&
                  specs[
                    (node.children[i + 1] as { panelId: PanelId }).panelId
                  ]?.pinned === true
                ) ? (
                  <div
                    role="separator"
                    aria-orientation={node.dir === "row" ? "vertical" : "horizontal"}
                    data-testid={`handle-${pathKey}-${i}`}
                    data-orientation={node.dir === "row" ? "vertical" : "horizontal"}
                    className={styles.handle}
                    onPointerDown={(e) => onHandlePointerDown(i, e)}
                  />
                ) : null}
              </div>
            );
          })}
        </div>
      );
    }

    return (
      <main
        data-testid="layout-engine"
        className={styles.engine}
        data-maximized={state.maximized ?? ""}
      >
        {renderNode(state.root, [])}
      </main>
    );
  }
  ```
