# RN Expo SDK 55 → 57 Upgrade + EAS Distribution Wiring — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Nature of this plan:** this is a *dependency upgrade*, not a feature build. There is no "write a failing test first" — you cannot write a failing unit test for "the SDK is 57". Each task is a **run → verify → commit** cycle whose gate is the *existing* test suite / `expo-doctor` / lint gauntlet passing under the new SDK. Treat a red gate exactly as you would a failing TDD test: stop, diagnose, fix, re-run.

**Goal:** Upgrade `@rtc/client-react-native` from Expo SDK 55 / RN 0.83 to SDK 57 / RN 0.86 and wire free-path EAS distribution, so the app runs in Expo Go (and via `eas go` during the SDK-57 rollout gap) with no paid Apple Developer account.

**Architecture:** Let `expo install --fix` drive version resolution; hand-fix only the four seams the tooling cannot see (the `@expo/metro-runtime` pnpm override, `jest-expo`, `babel-preset-expo`, `@testing-library/react-native`). Add `eas.json` (development + preview-APK profiles), the deferred `app.config.ts` projectId, and `expo-dev-client`. One worktree, one PR.

**Tech Stack:** Expo SDK 57, React Native 0.86, React 19.2, pnpm 11 workspaces + Turborepo, EAS Build, jest-expo + vitest.
