# Phase 2: HUD Shell Chrome — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the HUD shell **frame** that wraps the (Phase-1) workspace: a canvas boot sequence with three sequentially-cycled variants, a session-lock screen, the header chrome (account / language / notifications / theme picker / env badge), the bottom status bar, and the preferences modal. Match the prototype's look/motion/layout (`docs/design/dev-handoff/prototype/source/Reactive Trader.dc.html`) while keeping the real app's data and the clean-architecture grain. **Zero new domain code** — all orchestration is presentation reshaping over existing streams plus the Phase-0 preferences seams.

**Architecture:** Neutral orchestration lives in app-layer presenters/machines (`BootSequenceMachine`); dumb React components in `src/ui/shell/{boot,lock,chrome,status,prefs}` consume them only through `useHooks()`. The ONE allowed exception to the no-`setTimeout`/`setInterval` dumb-UI gate is the boot `<canvas>` `requestAnimationFrame` loop — `rAF` is **rendering**, not business/transport/persistence, so it stays in the VIEW (the prototype's `_drawBoot`/`_drawBootLaser`/`_drawBootDocking` port verbatim). Boot variant cycling, progress, and `onDone` are deterministic-under-fake-timers machine concerns (precedent: `RfqCountdownMachine` — progress derived from the timer **tick index**, never `Date.now()`). Variant persistence goes through the Phase-0 `PreferencesPort` (NEVER direct `localStorage` in UI). Connection/session readouts wire to the real `ConnectionStatusPresenter`; cosmetic readouts (FPS, build, biometric line, env badge) ship as explicitly-marked decorative components with NO port.

**Tech Stack:** TypeScript · React 19 (dumb components, React Compiler) · RxJS + `@rx-state/core` (machines) · `@react-rxjs/core` `bind` (global presenters) · Canvas 2D (boot, verbatim port) · Motion One via `#/ui/shell/motion/` (Phase-0 wrapper) for the cross-fade · CSS Modules + semantic `data-*` + `var(--token)` · Vitest (unit + contract) · Playwright-CT / Playwright / vitest-browser (visual goldens).

### Task 1 — `BootSequenceMachine` (neutral, deterministic, persisted variant cycling)

The boot machine is the spine of the boot sequence: it owns the 0→100 progress ramp over `DUR ≈ 4200ms`, the `onDone` hand-off, and the three-variant cycle persisted via the preferences seam (NOT `localStorage` in UI). Progress is derived from the timer **tick index** (RfqCountdownMachine idiom) so it is exact under `vi.useFakeTimers()`. Built before any UI so the view has a state contract to consume.

- [ ] **Step 1.3 — Minimal impl: the ramp + done.** Create `BootSequenceMachine.ts` deriving progress from the tick index exactly like `RfqCountdownMachine` derives `remaining` (no `Date.now()` in the stream math):
  ```typescript
  import { type StateObservable, state } from "@rx-state/core";
  import { concat, merge, of, Subject, timer } from "rxjs";
  import { map, takeUntil, takeWhile } from "rxjs/operators";

  import type { Machine } from "./machine";

  export type BootVariant = "core" | "laser" | "docking";
  export const BOOT_VARIANTS: readonly BootVariant[] = ["core", "laser", "docking"];
  export const BOOT_DURATION_MS = 4200;
  const BOOT_TICK_MS = 90;

  export interface BootSequenceState {
    readonly variant: BootVariant;
    readonly progress: number;
    readonly done: boolean;
  }
  export interface BootSequenceIntents { skip: () => void; }
  export interface BootSequenceDeps {
    readonly variant: BootVariant;
    readonly advance: (next: BootVariant) => void;
    readonly onDone: () => void;
  }

  const TICKS = Math.ceil(BOOT_DURATION_MS / BOOT_TICK_MS);

  export function createBootSequenceMachine(
    deps: BootSequenceDeps,
  ): Machine<BootSequenceState, BootSequenceIntents> {
    const variant = deps.variant;
    // Advance the persisted cycle pointer immediately, like the prototype does at
    // boot start (Reactive Trader.dc.html:846) — next run gets the next variant.
    const nextIdx = (BOOT_VARIANTS.indexOf(variant) + 1) % BOOT_VARIANTS.length;
    deps.advance(BOOT_VARIANTS[nextIdx]);

    const skip$ = new Subject<void>();
    const initial: BootSequenceState = { variant, progress: 0, done: false };

    // Progress derived from tick index i: pct = min(100, round(i / TICKS * 100)).
    // Deterministic under fake timers — no Date.now() in the math.
    const ramp$ = timer(0, BOOT_TICK_MS).pipe(
      map((i): BootSequenceState => {
        const progress = Math.min(100, Math.round((i / TICKS) * 100));
        return { variant, progress, done: progress >= 100 };
      }),
      takeWhile((s) => !s.done, true), // inclusive: emit the done:true tick, then complete
    );

    const skipped$ = skip$.pipe(
      map((): BootSequenceState => ({ variant, progress: 100, done: true })),
    );

    // The first of (ramp completion | skip) wins; takeUntil(skip$) cuts the ramp.
    const stream$ = merge(ramp$.pipe(takeUntil(skip$)), skipped$);

    const state$: StateObservable<BootSequenceState> = state(stream$, initial);

    // onDone fires exactly once when a done:true state lands.
    const doneSub = state$.subscribe((s) => {
      if (s.done) deps.onDone();
    });
    const warm = state$.subscribe();

    return {
      state$,
      intents: { skip: () => skip$.next() },
      dispose: () => {
        skip$.complete();
        doneSub.unsubscribe();
        warm.unsubscribe();
      },
    };
  }
  ```
  > `onDone` must fire once: `done:true` is emitted exactly once (ramp `takeWhile(inclusive)` then completes, or `skip$` emits a single `100/done` and the ramp is cut by `takeUntil(skip$)`). The `doneSub` observer calls `deps.onDone()` on that single emission.

  import { useEffect, useRef } from "react";

  import { useHooks } from "#/ui/hooks/useHooks";

  import {
    type BootDrawCtx,
    drawBootCore,
    drawBootDocking,
    drawBootLaser,
  } from "./bootCanvas";
  import styles from "./BootSequence.module.css";

  const DRAW = {
    core: drawBootCore,
    laser: drawBootLaser,
    docking: drawBootDocking,
  } as const;

  export function BootSequence({ onDone }: { onDone: () => void }): ReactElement | null {
    const { useBootSequence } = useHooks();
    const { state, skip } = useBootSequence(onDone);
    const canvasRef = useRef<HTMLCanvasElement | null>(null);

    useEffect(() => {
      const canvas = canvasRef.current;
      if (!canvas) return;
      const reduce = window.matchMedia?.("(prefers-reduced-motion: reduce)").matches;
      if (reduce) return; // skip the animated loop; chrome + cross-fade still apply
      const ctx = canvas.getContext("2d");
      if (!ctx) return; // jsdom / no-2d: render chrome only
      const cs = getComputedStyle(document.documentElement);
      const d: BootDrawCtx = {
        canvas, ctx, start: performance.now(),
        accent: cs.getPropertyValue("--accent-primary").trim(),
        accent2: cs.getPropertyValue("--accent-2").trim(),
        buy: cs.getPropertyValue("--accent-positive").trim(),
        sell: cs.getPropertyValue("--accent-negative").trim(),
      };
      const draw = DRAW[state.variant];
      let raf = 0;
      const loop = (): void => { draw(d); raf = requestAnimationFrame(loop); };
      loop();
      return () => cancelAnimationFrame(raf);
    }, [state.variant]);

    return (
      <div
        data-testid="boot-sequence"
        data-done={state.done ? "true" : "false"}
        data-variant={state.variant}
        className={styles.boot}
      >
        <canvas ref={canvasRef} className={styles.canvas} />
        <div className={styles.panel}>
          <div className={styles.wordmark}>REACTIVE&nbsp;TRADER</div>
          <div className={styles.subtitle}>TACTICAL TRADING OPERATING SYSTEM · v4.0</div>
          <div className={styles.progressRow}>
            <div className={styles.bar}>
              <div className={styles.fill} style={{ "--boot-pct": `${state.progress}%` } as CSSProperties} />
            </div>
            <span data-testid="boot-pct" className={styles.pct}>{state.progress}%</span>
          </div>
          <button type="button" data-testid="boot-skip" className={styles.skip} onClick={skip}>
            SKIP ▸
          </button>
        </div>
      </div>
