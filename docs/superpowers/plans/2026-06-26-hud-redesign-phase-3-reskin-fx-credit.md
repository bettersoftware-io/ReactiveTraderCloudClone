# Phase 3: Reskin FX + Credit — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform every existing `fx` and `credit` UI module to the HUD aesthetic — glass panels (`var(--panel)` / `var(--panel-blur)`), display + mono typography (`var(--font-display)` / `var(--font-mono)`), glow (`var(--glow)`), and the layered border system (`var(--border)` / `var(--border-strong)`) — while changing **zero** behavior. Price-tick digit flashes, fill/reject pulses, RFQ/quote-expiry pulses, and blotter new-row entrances are driven by the existing RxJS domain streams via the `AnimationDirector` seam (Phase 0), surfaced through `useAnimationIntents` → a `data-anim` attribute → CSS keyframes. An ambient aurora / grid-drift background (pure CSS) is gated by `useAnimatedBackground` and is off by default. All existing contract specs stay green (behavior unchanged); only the **visual** goldens are re-baselined for the new look.

**Architecture:** Dumb React over presenters/machines via `useHooks()` only. Phase 3 adds **no domain code** and **no markup-logic changes** beyond (a) new semantic `data-*` state hooks and additional `className`s, and (b) reading `useAnimationIntents(target)` to set a `data-anim` attribute. All styling is CSS Modules + semantic `data-*` selectors + `var(--token)`; the only inline style permitted remains a **computed `--custom-property` geometry value** (precedent: `RfqCountdown` sets `style={{ "--rfq-fill": ... }}`). New token consumption assumes Phase 0 has shipped the extended `ThemeTokens` (interfaces §3), `AnimationDirector` + `useAnimationIntents` (interfaces §4), and the `animatedBackground$` preference (interfaces §2). Phase 1 has placed these module roots inside the layout panels; Phase 3 only restyles their internals.

**Tech Stack:** React 19 + CSS Modules; RxJS streams behind presenters; Motion One wrapper at `#/ui/shell/motion/` (Phase 0) is the only animation-dependency import site — Phase 3 uses **CSS keyframes** driven by `data-anim`, not Motion One directly, except where an orchestrated entrance is wanted. Visual goldens: Playwright-CT (`test:ui:visual:playwright-ct:react`), plain Playwright (`test:ui:visual:playwright:react`), and vitest-browser (`test:ui:visual:vitest-browser:react`). Contract specs: vitest sociable RTL (`test:ui:contract`).

Shared interfaces: see [`2026-06-26-hud-redesign-interfaces.md`](./2026-06-26-hud-redesign-interfaces.md). The following bind every task in this plan (verbatim from the interfaces doc's "Cross-plan global constraints"):

**Phase-3-specific invariants:**
- **Behavior unchanged.** No change to any `.tsx` control flow, event handler, presenter, machine, or domain stream. If a change would alter what a contract spec asserts, it is out of scope — stop and reconsider.
- **Contract specs are frozen** — they must pass **unmodified** after each task (they assert behavior, structure, and text, not pixels). The TDD signal for restyle tasks is therefore: contract spec **green before and after**; the visual golden is what changes.
