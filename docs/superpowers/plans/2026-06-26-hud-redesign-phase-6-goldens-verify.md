# Phase 6: Goldens & Verify — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the entire HUD redesign (Phases 0–5) green on **both** committed visual-golden sets and **every** gate. Regenerate the local `react-local/<platform>-<arch>/` goldens across all three visual tiers, drive the CI `react/` (x86) canonical set via the GitHub Actions workflow, re-baseline the contract specs whose assertions legitimately changed, run the full local verification gauntlet plus the 10-suite e2e (including the new equities + admin suites), do a final dumb-UI gate audit + CSS-modules/tokens sweep, and refresh the coverage reports. This is the consolidation phase: it adds **zero** new product behaviour — it confirms and records.

**Architecture:** Pure verification + golden regeneration. The neutral contract/shared layers (`tests/ui/visual/shared/scenarios.ts` + `fixtures.ts`, the `react/` swap-trio) and the framework-portability contract are untouched in shape; only the committed PNG baselines and any legitimately-changed contract assertions move. The dual-golden routing (`process.env.CI ? "react" : "react-local/${os.platform()}-${os.arch()}"`) and dumb-UI grep-gates 26–29 are the load-bearing invariants this phase re-asserts.

**Tech Stack:** pnpm 11 workspaces + Turborepo; Vitest 4 (unit / app / contract / vitest-browser visual tier); Playwright 1.61 (`@playwright/test` plain tier + `@playwright/experimental-ct-react` component tier); Biome 2.5.0 (format + lint), ESLint 10 (AST + type-aware), stylelint 17 (`declaration-strict-value`), manypkg + syncpack, knip, dependency-cruiser; istanbul + v8 coverage; the `update-visual-goldens.yml` GitHub Actions workflow renders the canonical x86 set inside `mcr.microsoft.com/playwright:v1.61.0-noble`.
